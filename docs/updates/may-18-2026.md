#### Tracking & Build Update Report ⸬ Mon May 18, 2026  09:25:00 am - +08:00

---

## 1. Meta Pixel Integration
- Meta Pixel is now integrated directly into the app runtime and is no longer hardcoded to a single dataset id inside the component.
- The pixel id is now read from the existing admin setting so it can be changed without editing code.
- The integration is mounted safely inside the provider tree and supports App Router navigation, so page views are no longer limited to the first page load only.

- Update:
The Meta Pixel admin settings form is now aligned with the frontend integration. It now reads and writes the same key properly, reflects the saved value correctly, and can be used as the source of truth for the live pixel setup.

---

## 2. Add To Cart Tracking
- `AddToCart` tracking is now wired into the shared cart add flows instead of only one UI surface.
- This means product cards, the product detail page, recommendations, cart history re-adds, and authenticated bundle adds now report consistently through the same path.
- The tracking is fired only after the add flow succeeds, which avoids reporting failed add attempts as successful conversions.

---

## 3. Checkout Tracking
- I added a custom `CheckoutClick` event for checkout CTA interactions so we can see where checkout intent is coming from.
- The cart drawer checkout CTA now reports its own click source before routing to the cart page.
- The cart page checkout CTA now also reports standard `InitiateCheckout` with item count, product ids, and cart value.

- Notes:
This setup separates "user clicked checkout" from "user actually initiated checkout", which makes funnel analysis cleaner and easier to debug later.

---

## 4. Shop Now CTA
- The hero `Shop Now` CTA on the home page is now tracked.
- It emits a `ShopNowClick` event with the CTA label, href, and location marker so we can identify this button separately from the rest of the navigation.

---

## 5. Navbar Menu Tracking
- The radial navbar menu now reports `NavigationClick` events.
- This covers both the main category links and the subitems inside the category viewport.
- Category slug, href, label, and click location are included so menu engagement can be segmented more precisely.

---

## 6. Product Page View Tracking
- Product detail pages now emit standard `ViewContent`.
- The event includes the product id, product name, category, and the default displayed price derived from the same pricing logic used by the UI.
- The event is guarded so it only fires once per viewed product state instead of spamming on every render.

---

## 7. Build Warning Resolution
- The deprecation warnings:
`[DEP0205] DeprecationWarning: module.register() is deprecated. Use module.registerHooks() instead.`
were not coming from app code.
- They were coming from `@tailwindcss/node` during the Next.js/Turbopack build under Node `v26.0.0`.

- Update:
I added a persistent postinstall patch that rewrites Tailwind's loader bootstrap to use `module.registerHooks()` when the runtime supports it, while keeping the older fallback path for runtimes that still need `module.register()`.

- Result:
The 3 `DEP0205` warnings no longer appear in traced production builds.
